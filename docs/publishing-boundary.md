# Publishing Boundary

This document defines what belongs in the public showcase repository and what stays private.

## Publish Directly

- visual primitives
- chart helpers
- palette tokens
- demo-safe fixtures
- high-level architecture notes
- public analyzer contract documentation

## Publish After Adaptation

- demo shells derived from app navigation
- simplified diagrams of the analyzer boundary
- screenshots and motion clips that do not reveal unreleased work

## Keep Private

- production analyzer runtime
- private model assets
- training data and training code
- billing, auth, backend, and operational code
- internal experiments and unreleased product work

## Working Rule

If a file helps someone understand Voiceland's product quality or engineering discipline without materially helping them reproduce the core product capability, it is a candidate for this repo.
